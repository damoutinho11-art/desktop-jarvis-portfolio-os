# J.A.R.V.I.S. v23.0 Crypto-Lane Public Signal Selection Gate

## Purpose

v23 turns the v22 multi-crypto public data quality pipeline into a crypto-lane selection gate.

It combines:

- BTC/HYPE/TAO source-quality readiness from v22
- current platform readiness
- current allocation/risk/budget output
- crypto-lane executable amounts

The result is a selected crypto candidate for evidence/scoring integration.

## Scope

Included:

- selection among BTC/HYPE/TAO seed crypto candidates
- source-quality readiness check
- platform readiness check
- executable amount check from current allocation/risk output
- safety flags proving no mutation or execution

Excluded:

- approval ticket mutation
- allocation scoring mutation
- buy request creation
- broker connection
- credentials
- private account ingestion
- order placement
- trade execution

## Product rule

v23 is not the final crypto universe. BTC/HYPE/TAO are the seed universe. Later stages should add a crypto candidate registry and expand data coverage, but the selection gate must exist first so expansion remains rule-based instead of random.
